## 我能实现同一个镜像，然后动态地去传入这个环境变量吗？
「编译时硬编码」改为「运行时注入」

开发时留占位符 → 构建时打通用镜像（带替换工具） → 运行时传真实变量 → 脚本替换占位符 → 前端用真实值。
```
【开发阶段】→【构建镜像阶段】→【运行容器阶段】→【前端最终运行】
  写占位符    造“空白盒子”     填真实信息       用填好的信息
```

```js
// 
开发阶段：前端代码写占位符（如__API_URL__）	

构建镜像阶段：编译前端代码 + 打包 Nginx + 替换脚本	

运行容器阶段：docker run -e 变量=值 + 脚本替换	

前端最终运行：前端读取替换后的真实值	
```

```js
# 脚本要做的事：
1. 读取你通过 docker run -e 传入的 API_URL 值（比如 "https://prod-api.com"）；
2. 找到前端所有 JS 文件里的 "process.env.API_URL" 这个字符串；
3. 把这个字符串替换成 "\"https://prod-api.com\""（加引号是为了让 JS 识别为字符串）；
4. 保存文件，然后启动 Nginx 提供服务。
```
